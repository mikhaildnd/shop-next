# Architecture

## Components

Компоненты располагаются максимально близко к месту использования.

Используются три уровня размещения:

- page-specific компоненты;
- domain components;
- глобально переиспользуемые компоненты.

Page-specific компоненты располагаются рядом с `page.tsx`.

Компоненты, используемые несколькими страницами одного домена, могут располагаться в общей папке `_components`.

Глобально переиспользуемые компоненты располагаются в `components/`.

При наличии нескольких вариантов отображения рекомендуется отделять layout от содержимого. Layout-компоненты отвечают только за способ отображения и принимают содержимое через `children`, а композиция элементов инкапсулируется отдельным компонентом.

---

## Hooks

Переиспользуемые хуки располагаются в `hooks/`.

Feature-хуки могут располагаться рядом со своим доменом.

---

## Services

Доменная логика располагается в `services`.

Типы, мапперы и дополнительные файлы располагаются рядом со своим доменом.

Дополнительные слои (`use-cases` и т.п.) вводятся при необходимости.

---

## Types

Типы располагаются рядом со своим доменом.

Общие типы проекта могут располагаться в `src/types`, если они используются несколькими доменами.

Типы, относящиеся только к одной feature, следует располагать рядом со своим доменом, а не в `src/types`.

---

## lib

`lib` содержит чистые функции и константы.

Код в `lib` не зависит от:

- React;
- JSX;
- базы данных.

Структура организуется по функциональным областям.

Следует избегать неопределённых глобальных папок (`helpers`, `common`).

---

## Routes

Маршруты инкапсулируются в `routes.ts`.

Page routes и API routes разделяются.

Построение URL выполняется через специализированные функции и шаблонные строки.
