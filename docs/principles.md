# Principles

## Основные принципы

Проект развивается постепенно.

Приоритеты:

1. Простота.
2. Предсказуемость.
3. Единый стиль.
4. Маленькие коммиты.
5. Небольшие этапы рефакторинга.

## Подход

Предпочитать composition over abstraction.

Не делать рефакторинг ради рефакторинга.

Не создавать дополнительные абстракции без явной пользы.

Избегать функций-обёрток над одной строкой.

Держать связанные сущности рядом.

Оптимизировать читаемость раньше, чем "идеальную архитектуру".

Предпочитать небольшие специализированные функции.

Рефакторинг выполнять постепенно, небольшими этапами.

## Documentation

Документация является частью проекта.

При изменении архитектуры, структуры или значимых решений необходимо поддерживать документацию в актуальном состоянии.

Предпочитать обновление существующих документов вместо создания новых.

### current-state.md

Обновлять, если изменилось текущее состояние проекта:

* появились новые домены;
* изменилась структура данных;
* появились новые механизмы поиска, фильтрации или пагинации;
* добавлены новые крупные возможности.

### progress.md

Обновлять после завершения крупных этапов проекта, например:

* аутентификация;
* корзина;
* избранное;
* отзывы;
* оформление заказа;
* админ-панель;
* платежи.

Не добавлять записи после каждого коммита.

### Architecture и Style Guide

При появлении новых соглашений или архитектурных решений необходимо отражать их в документации.

Во время работы над проектом следует оценивать, требует ли изменение обновления документации.
